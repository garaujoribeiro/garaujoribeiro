  ## What I Care About

  I like systems that are straightforward to reason about in production:

  - APIs with clear contracts and predictable failure modes
  - logs and metrics that explain what actually happened
  - automation that can recover, retry, and report useful state
  - infrastructure that is boring enough to trust
  - code that keeps business rules explicit instead of hiding them in cleverness

  The work I enjoy most sits between backend engineering and systems programming: services that coordinate real workflows, bots that interact with messy external platforms, and
  infrastructure that makes those systems observable and operable.
