  Backend and systems-oriented developer working mostly on Java services, automation infrastructure, and lower-level integrations.

  Most of my current work is private and tied to Match Sales: backend systems, user and business services, WhatsApp automation, meeting bots, AWS infrastructure, observability, and the kind
  of production glue that does not look glamorous but has to be correct.

  I still write TypeScript, but mainly where it makes sense: bots, automation, webhooks, SDKs, and service tooling. I use Rust when the work gets closer to system boundaries, especially
  around Zoom automation with the Linux SDK.

  I like systems that are straightforward to reason about in production:

  - APIs with clear contracts and predictable failure modes
  - logs and metrics that explain what actually happened
  - automation that can recover, retry, and report useful state
  - infrastructure that is boring enough to trust
  - code that keeps business rules explicit instead of hiding them in cleverness

  The work I enjoy most sits between backend engineering and systems programming: services that coordinate real workflows, bots that interact with messy external platforms, and
  infrastructure that makes those systems observable and operable.

  I believe steady work compounds. Build carefully, learn honestly, ship what helps, and keep going.
